# Good First Issue Candidates

These are small, bounded contributor tasks maintainers can promote into GitHub issues with the `good first issue` label. Confirm that a candidate is still open before starting it.

1. Add a unit test covering malformed `--bridge-url` CLI input without changing runtime behavior.
2. Add accessible alt-text checks for images referenced by the generated Pages landing page.
3. Convert one remaining pure-Python smoke script into a `unittest` module while retaining the original release gate until parity is proven.
4. Improve one client guide's troubleshooting section using a reproducible, non-secret failure example.
5. Add a read-only registry report that summarizes tool counts by owner, group, and exposure.

Each issue should name its owner test, explicitly exclude safety-contract changes, and link to [Contributing](../CONTRIBUTING.md) and [Adding a Tool](ADDING_A_TOOL.md) when relevant.
